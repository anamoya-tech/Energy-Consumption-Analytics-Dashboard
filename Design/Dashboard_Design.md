# 🎨 Dashboard Design – Energy Consumption

## 🧠 Design Philosophy

This dashboard is built with a clear principle:

> 👉 Prioritizing decision-making over aesthetics

Every visual element has a specific purpose:
- provide information quickly  
- enable diagnosis  
- guide actions  

❌ Not a decorative dashboard  
✅ A decision-making tool  

---

## 🏗️ Visual Structure

The dashboard follows an analytical app structure divided into three pages:

### 🟢 Overview → 🟡 Diagnosis → 🔴 Action

👉 Natural user flow:
1. Understand what is happening  
2. Identify the problem  
3. Take action  

---

## 📐 Layout & Composition

### Grid and Structure
- Grid-based layout  
- Strict component alignment  
- Consistent spacing  

---

### Visual Density
- Maximum 5–6 visuals per page  
- Removal of redundant elements  
- Use of whitespace as a design element  

👉 Result: clear understanding in under 10 seconds  

---

## 🎯 Visual Hierarchy

The interface is designed to guide attention:

1. **Main KPIs (top section)**  
2. **Context chart (trend)**  
3. **Detailed analysis**  
4. **Action table**  

### Applied Rules:
- Size = importance  
- Color = meaning  
- Position = priority  

---

## 📊 KPI Design

### Objective
Make KPIs instantly understandable

---

### Implementation
- Custom KPI cards using HTML 【1-7f0f03】  
- Product-style design (not standard Power BI cards)  
- Each KPI includes:
  - main value  
  - context  
  - visual indicator (positive / negative)  

---

### Result

👉 More readable and impactful KPIs  

---

## 🎨 Color System

### Palette
- Primary color → dashboard identity  
- Green → positive  
- Red → alert  
- Grey → secondary elements  

---

### Principles
- Minimal use of colors  
- Consistency across all pages  
- Color only when it adds meaning  

---

## 🔍 Visual Selection

Only visuals that provide value are used:

✅ Custom KPI cards  
✅ Line chart → trends  
✅ Bar chart → comparisons  
✅ Table → actions  

---

❌ Removed:
- unnecessary charts  
- decorative elements  
- duplicate visuals  

---

## 🧭 Navigation

The dashboard behaves like an application:

- Guided flow between pages  
- Global filters (date, plant, equipment)  
- Consistent interactions between visuals  

👉 Prevents user confusion  

---

## ⚙️ Visual Consistency

Consistency is maintained across:

- Typography  
- Colors  
- Spacing  
- Number formatting  
- Chart styles  

👉 Reduces cognitive load  

---

## 🚫 Design Mistakes Avoided

- KPIs without context  
- Overloaded views  
- Excessive use of colors  
- PowerPoint-style dashboards  
- Lack of visual hierarchy  

👉 Every element answers a specific question  

---

## 👁️ User Experience (UX)

The dashboard is designed to:

- be understood in seconds  
- require no explanation  
- guide the user naturally  

### Core Principle

> “If you need to explain how to use it, the design has failed”

---

## 🧪 Key Design Decisions

- Use of HTML for full design control 【1-7f0f03】  
- Clarity prioritized over complexity  
- Focus on analysis, not decoration  

---

## 📈 Design Outcome

✔️ High readability  
✔️ Low cognitive effort  
✔️ Intuitive navigation  
✔️ Decision-oriented  

---

## 🚀 Future Improvements

👉 Add dynamic visual indicators:
- KPI alerts  
- color changes based on deviations  
- dynamic insight text  

---

## 🔚 Conclusion

> A great dashboard is not the most visually appealing one, but the one that enables better and faster decisions.
